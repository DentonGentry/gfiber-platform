Two files are read by diagd:
1) diag_kern_err_msgs.txt - list the kernel critical and error messages 
   monitored by diagd
2) diag_kern_warn_msgs.txt- list the kernel warning message monitored by diagd
3) NOTE - For performance issue, do not change tokens order in the example.
          diagd is always assumed the token order same as example.

Each line format in the files:

 Token     Descriptions
================================================================================
 dtoken - 0: indicate there is no token in the dkmsg string.
          1: indicate there is at least one token in the dkmsg string.
          dtoken string is @@@
 dact   - log:  indicate log the message
          hwerr: indicate it's a hardware releated issue, log the flag to inform user.
 msglvl - specify diagd message levels:
          0 - critical error
          1 - significant error
          2 - software error
          3 - warning
          4 - info
 code   - indicate error code and its value is 4 digits of hex number.
 dkmsg  - follw by the kernel string to be compared.

Examples:

 dtoken=0 dact=hwerr msglvl=1 code=0001 dkmsg=moca_probe: can't get resources
 dtoken=1 dact=log msglvl=2 code=0101 dkmsg=bcmgenet_xmit: tx ring @@@ full when queue awake
